# ksiegai-next Agent Guide

> Source of truth for work in `ksiegai-next/`, aligned to the current implementation.

## 1. What This Project Is

- App: `ksiegai-next` (marketing + auth handoff site)
- Stack: Next.js 14 App Router + TypeScript + Supabase Auth
- Entry/layout: `app/layout.tsx`
- Home page: `app/page.tsx`
- Dev server: `next dev` (default port `3000`)
- Build mode: static export (`output: "export"`, `trailingSlash: true`) via `next.config.js`

## 2. Non-Negotiables

1. SEO metadata quality is mandatory for public pages.
2. Cross-domain auth token contract with app must remain compatible.
3. Do not move AB-test runtime to direct DB calls from browser; keep static file / API flow.
4. Use env vars/secrets for sensitive keys; do not hardcode credentials.

## 3. Real Route Surface

Observed major routes in `app/`:
- Marketing: `/`, `/cennik`, `/dla-ksiegowych`, `/jak-to-dziala`, `/premium`, `/governance`, `/infrastructure`
- Legal: `/regulamin`, `/polityka-prywatnosci`, `/polityka-zwrotow`, `/rodo`
- Auth pages: `/rejestracja`, `/logowanie`, `/auth/login`, `/auth/callback`, `/auth/confirm`
- Technical: `/api/ab-track`, `/robots`, `/sitemap`, `/manifest`

Use existing Polish URL conventions where already established.

## 4. Auth Handoff Contract (Outbound)

- Helper: `lib/auth/crossDomainAuth.ts`
- Token key: `ksiegai_auth_token`
- Store in both cookie + localStorage
- Redirect target:
  - local dev: `http://localhost:8080`
  - prod: `https://app.ksiegai.pl`

Auth callback (`app/auth/callback/page.tsx`) exchanges code, stores token, then redirects to app.

## 5. AB Testing Architecture (Current)

- Prebuild fetch script: `scripts/fetch-ab-tests.js`
  - writes `public/ab-tests.json`
- Homepage reads cached/static AB payload (`useABTestSSG`, inline preload script in `app/page.tsx`).
- Event/assignment tracking endpoint: `app/api/ab-track/route.ts`
  - uses Supabase service-role credentials from env.
- Current DB entities used by code: `ab_test_definitions`, `ab_test_assignments`, `ab_test_events`.

## 6. SEO and Tracking

- Global metadata is defined in `app/layout.tsx`.
- Robots and sitemap are generated by `app/robots.ts` and `app/sitemap.ts`.
- GTM snippets are present in layout and homepage logic.

Any page additions should include:
- clear title/description
- canonical awareness
- no accidental indexing restrictions (this is public site)

## 7. Practical Rules for Changes

When adding a public page:
1. add route file in `app/`
2. ensure metadata is present
3. include in sitemap if needed
4. confirm static export compatibility (no unsupported runtime-only patterns)

When changing auth flow:
1. preserve token schema compatibility with `ksef-ai`
2. verify localhost and production redirect behavior
3. verify callback path behavior for OAuth/magic-link

When changing AB testing:
1. keep prebuild output contract (`public/ab-tests.json`)
2. keep API event payload backward-compatible
3. verify homepage variant preload fallback still works without runtime DB dependency

## 8. Known Sharp Edges

- Some auth pages are duplicated by path style (`/logowanie` and `/auth/login`); avoid removing paths without checking external links.
- Static export constraints apply to routing/features.
- Some workspace docs still reference old route names and older AB table names.
